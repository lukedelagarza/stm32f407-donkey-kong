# CLAUDE.md

## Project Overview

Bare-metal C firmware for the **STM32F407VG** (ARM Cortex-M4F) implementing a **Donkey Kong arcade game**. No HAL, no RTOS — direct register access via CMSIS. Target board: STM32F4DISCOVERY.

Current state: early scaffold. `src/main.c` is a skeleton; the game (planned FSM, SPI display driver, input handling, collision detection) is yet to be built.

Peripheral reference: `RM0090`; see `docs/README.md` for the document manifest.

## Commands

```sh
make          # build ELF/BIN/HEX into build/, then print size
make clean    # remove build/
make size     # section sizes (text/data/bss)
make list     # interleaved source+asm listing (build/firmware.lst)
make dump     # full ELF disassembly piped to less
make help     # list all targets
make V=1      # verbose: echo full compiler/linker commands

make flash         # program via OpenOCD + ST-LINK (preferred)
make flash-stlink  # program via st-flash
make debug         # OpenOCD GDB server on :3333 + attach arm-none-eabi-gdb
```

Requires `openocd` and `arm-none-eabi-gdb` (or `st-flash`) on PATH.

## Toolchain

`arm-none-eabi-gcc`, C11, `-mcpu=cortex-m4 -mthumb -mfpu=fpv4-sp-d16 -mfloat-abi=hard`, `-O0 -g3`. Warnings: `-Wall -Wextra -Wshadow -Wdouble-promotion -Wformat=2 -Wundef`. LSP: `clangd` via `compile_commands.json` (gitignored, generated by `bear`).

## Code Structure

| Path | Purpose |
|------|---------|
| `src/main.c` | Application entry point — build firmware here |
| `inc/` | Project headers (empty; add as needed) |
| `cmsis/` | CMSIS core + STM32F4xx register-map headers |
| `startup_stm32f407vgtx.s` | Reset handler, vector table, stack/heap init |
| `system_stm32f4xx.c` | `SystemInit()` — clock setup, runs before `main()` |
| `syscalls.c` / `sysmem.c` | Newlib syscall stubs and heap |
| `STM32F407VGTX_FLASH.ld` | Linker script (1 MB FLASH, 128 KB RAM, 64 KB CCMRAM) |

## Memory Map & Constraints

- FLASH `0x08000000` (1 MB), RAM `0x20000000` (128 KB), CCMRAM `0x10000000` (64 KB), peripherals `0x40000000+`.
- **CCMRAM is not DMA-accessible** — keep DMA buffers in SRAM.
- `nano.specs` + `nosys.specs`: newlib-nano with stub syscalls; `printf` needs SWO/semihosting wired up explicitly.
- `-ffunction-sections -fdata-sections` + `--gc-sections`: unused code/data stripped at link.
